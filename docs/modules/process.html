Module process

The process module allows querying various properties about the current process, as well as creating, modifying, and searching other processes.

Functions

getpid () Returns the process ID of the current process.
getppid () Returns the process ID of the parent process, if available.
getuser () Returns the username the process is running under.
setuser (user) Sets the user of the current process.
clock () Returns the amount of time this process has executed.
getenv () Returns the environment table for the current process.
getname () Returns the name of the current process.
getcwd () Returns the working directory of the current process.
chdir (dir) Sets the working directory of the current process.
fork (func, name, ...) Creates a new process running the specified function with arguments.
exec (path, ...) Replaces the current process with the contents of the specified file.
newthread (func, ...) Creates a new thread running the specified function with arguments.
exit (code) Ends the current process immediately, stopping all threads and sending the specified return value to the parent.
getplist () Returns a list of all valid PIDs.
getpinfo (pid) Returns a table with various information about the specified process.


Functions

getpid ()
Returns the process ID of the current process.
getppid ()
Returns the process ID of the parent process, if available.
getuser ()
Returns the username the process is running under.
setuser (user)
Sets the user of the current process. This can only be run by root.

Parameters:

  • user string The user to switch to
clock ()
Returns the amount of time this process has executed. This may not be entirely accurate due to a lack of precision in the system clock.
getenv ()
Returns the environment table for the current process.
getname ()
Returns the name of the current process.
getcwd ()
Returns the working directory of the current process.
chdir (dir)
Sets the working directory of the current process.

Parameters:

  • dir string The new working directory, which must be absolute and existent.
fork (func, name, ...)
Creates a new process running the specified function with arguments.

Parameters:

  • func function The function to run in the new process. This will be the main function of the first thread, and will have its environment set to the new process's environment.
  • name string ? The name of the new process.
  • ... any Any arguments to pass to the function.

Returns:

    The PID of the new process.
exec (path, ...)
Replaces the current process with the contents of the specified file. This function does not return - it can only throw an error.

Parameters:

  • path string The path to the file to execute.
  • ... any Any arguments to pass to the file.
newthread (func, ...)
Creates a new thread running the specified function with arguments. Threads in the same process share the same environment, event queue, and other properties.

Parameters:

  • func
  • ...
exit (code)
Ends the current process immediately, stopping all threads and sending the specified return value to the parent. This function does not return.

Parameters:

  • code number ? The value to return.
getplist ()
Returns a list of all valid PIDs.
getpinfo (pid)
Returns a table with various information about the specified process.

Parameters:

  • pid number The process ID to query.

Returns:

    { id = number, name = string, user = string, parent? = number, dir = string, stdin? = number, stdout? = number, stderr? = number, cputime = number, systime = number, threads = { [number] = { id = number, name = string, status = string } } }|nil The process information, or nil if the process doesn't exist.
