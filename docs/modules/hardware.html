Module hardware

Functions

wrap (device)
Wraps a device into an indexable object, allowing accessing properties and methods of the device by indexing the table. If an object is passed, this simply re-wraps the device in a new object.

Parameters:

  • device string The device specifier or object to wrap

Returns:

    device The wrapped device

Usage:

    Wrap a device, use a property, and call a method:

    local computer = hardware.wrap("/")
    print(computer.isOn)
    computer.label = "My Computer"
    computer:reboot()

path (device)
Returns a list of device paths that match the device specifier or object. If an absolute path is specified, this returns the same path back. If a device object is specified, this returns the path to the device.

Parameters:

  • device string or device The device specifier or object to read

Returns:

    string... The paths that match the specifier or device object.

Calls a method on a device.

Returns:

listen (device[, state=true])
Toggles whether this process should receive events from the device.

Parameters:

  • device string or device The device specifier or object to modify
  • state boolean Whether to allow events (default true)
lock (device[, wait=true])
Locks the device from being called on or listened to by other processes.

Parameters:

  • device string or device The device specifier or object to modify
  • wait boolean Whether to wait for the device to unlock if it's currently locked by another process (default true)

Returns:

    boolean Whether the current process now owns the lock

See also:

Fields

tree
A table that allows accessing device object pointers in a tree. This is simply syntax sugar for real paths.

Usage:

    To access the left redstone signal

    local device = hardware.wrap(hardware.tree.redstone.left)
    print(device.input)
