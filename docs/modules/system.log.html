Module system.log

The log module exposes functions for interacting with the logging subsystem.

The default system log is available under the root `log` table. Other logs created through log.create can be accessed by indexing the `log` table with the name of the log, e.g. `log.mylog.info("hello")`. Both the main and subtables may also be called directly, e.g. `log("test")` or `log.mylog("hello")`.

Opens a log for listening to messages.
