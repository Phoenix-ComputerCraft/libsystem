Module system.hardware¶

The hardware module implements functions for operating on peripherals and other hardware devices.

API Reference¶

system.hardware.wrap(device: string): (result: device)¶

Wraps a device into an indexable object, allowing accessing properties and methods of the device by indexing the table.

If an object is passed, this simply re-wraps the device in a new object.

@usage Wrap a device, use a property, and call a method:

local computer = hardware.wrap("/")
print(computer.isOn)
computer.label = "My Computer"
computer:reboot()
Parameters:

device (string) – The device specifier or object to wrap

Returns:

result (device) – The wrapped device

system.hardware.find(type: string): (...: device)¶

Returns a list of wrapped devices that implement the specified type.

See :lua:obj:hardware.wrap: For wrapping a single device by path

Parameters:

type (string) – The type to search for

Returns:

... (device) – The devices found, or nil if none were found

system.hardware.path(device: string | device): (...: string)¶

Returns a list of device paths that match the device specifier or object.

If an absolute path is specified, this returns the same path back. If a device object is specified, this returns the path to the device.

Parameters:

device (string | device) – The device specifier or object to read

Returns:

... (string) – The paths that match the specifier or device object.

system.hardware.listen(device: string | device, state?: boolean): unknown¶

Toggles whether this process should receive events from the device.

Parameters:

system.hardware.lock(device: string | device, wait?: boolean): (result: boolean)¶

Locks the device from being called on or listened to by other processes.

it’s currently locked by another process

See :lua:obj:unlock: To unlock the device afterward

Parameters:

  • device (string | device) – The device specifier or object to modify

  • wait? (boolean) – Whether to wait for the device to unlock if (defaults to true)

Returns:

result (boolean) – Whether the current process now owns the lock

Parameters:

system.hardware.tree: table | nil¶

A table that allows accessing device object pointers in a tree. This is simply syntax sugar for real paths. @usage To access the left redstone signal

local device = hardware.wrap(hardware.tree.redstone.left)
print(device.input)
