Module system.serialization¶

The serialization module provides functions for serializing and deserializing objects in multiple formats, as well as some miscellaneous encoding types.

Index¶

Module

base64

Base64 encoder/decoder

json

JSON encoder/decoder

lua

Lua table encoder/decoder

toml

TOML configuration encoder/decoder

Function

struct()

Defines a structure coder from C structure code.

module system.serialization.toml¶

TOML configuration encoder/decoder

system.serialization.toml.encode(tbl: table, opts?: {indent: boolean}): (result: string)¶

Encodes a table into TOML format. This table must only have integer or string keys in itself and each subtable, and cannot mix strings and ints.

Parameters:

  • tbl (table) – The table to encode

  • opts? ({indent: boolean}) – Any options to specify while encoding

Returns:

result (string) – The encoded TOML data

system.serialization.toml.decode(str: string, opts?: {binary_escapes: boolean | nil}): (result: table)¶

Parses TOML data into a table.

Parameters:

  • str (string) – The TOML data to decode

  • opts? ({binary_escapes: boolean | nil}) – Options for decoding

Returns:

result (table) – A table representing the TOML data

system.serialization.toml.save(val: table, path: string, opts?: {indent: boolean})¶

Saves a table to a TOML file.

  • path (string) – The path to the file to save

system.serialization.struct(def: string, types?: table): (
    result: fun(obj: any, pos: number | nil): any, number | nil
)¶

Defines a structure coder from C structure code.

See https://gist.github.com/MCJack123/2e60f0b1c01411f4fe91d902212e33c9 for more information about how this works.

Parameters:

  • def (string) – The C code to compile into types

  • types? (table) – A table containing previously defined types; types will also be stored back into this table

Returns:

result (fun(obj: any, pos: number | nil): any, number | nil) – The coder for the last defined type, which can either take an object to encode (usually a table for a struct) and returns a string, or a string to decode and optionally a position to decode from and returns the decoded object + the next position to decode from
