Modules

system.expect The expect module provides error checking functions for other libraries.
system.filesystem The filesystem module implements common operations for working with the filesystem, including wrappers for syscalls.
system.framebuffer The framebuffer library provides functions to make "window" and "framebuffer" objects.
system.graphics The graphics module provides functions to draw primitive geometry on a locked terminal object.
system.hardware The hardware module implements functions for operating on peripherals and other hardware devices.
system.ipc The IPC module provides functions for sending messages to other processes.
system.keys The keys module assigns names to the keycode constants that Phoenix sends in key events, and adds a few functions to make using them easier.
system.log The log module exposes functions for interacting with the logging subsystem.
system.network The network module implements functions for making and hosting connections with local and Internet-connected computers, as well as managing the network stack configuration.
system.pretty Provides a "pretty printer", for rendering data structures in an aesthetically pleasing manner.
system.process The process module allows querying various properties about the current process, as well as creating, modifying, and searching other processes.
system.sync The sync library exposes interfaces for various synchronization structures.
system.terminal The terminal module defines functions to allow interacting with the terminal and screen, as well as handling user input.
system.util The util module contains various functions that don't have any specific system function, or help improve the usability of the general system.
